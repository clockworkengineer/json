# Library Development Guide

This guide provides instructions and best practices for developing and contributing to the `json_lib` library.

## Getting Started
- Clone the repository
- Navigate to the `library` directory
- Install Rust and Cargo
- Build the library using `cargo build` in the `library` folder
- Run library tests using `cargo test` in the `library` folder

## Branching Strategy
- Use feature branches for new library features
- Use bugfix branches for library fixes
- Merge changes via pull requests

## Code Style
- Follow Rust formatting conventions
- Document all public functions, modules, and types
- Prefer clear, explicit code over cleverness

## Testing
- Write unit and integration tests for all new library code
- Ensure all tests pass before submitting changes
- Use the `library/src/integration_tests` for integration scenarios

## Contributing
- Fork the repository
- Create a branch for your library changes
- Submit a pull request with a clear description of your changes and their impact on the library

## Contact
For questions about library development, open an issue or contact the maintainers via GitHub.
